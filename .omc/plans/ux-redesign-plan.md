# UX/UI 종합 개선 계획 — 옵티써치 페이지 통합 및 리디자인

**작성일:** 2026-04-05
**대상 페이지:** 저장된 키워드, 쇼핑인사이트, 대량분석, 키워드비교, AI도구

---

## 현황 분석 요약

### 전체 구조 문제
현재 사이드바에 7개 페이지가 나열되어 있고, 기능 간 연결이 거의 없음:
- `/analyze` (키워드 분석) → 단일 키워드 분석, 결과에서 다른 페이지로의 연결 없음
- `/keywords` (저장된 키워드) → 단순 목록, "분석" 버튼만 존재
- `/compare` (키워드 비교) → 키워드를 처음부터 수동 입력해야 함
- `/bulk` (대량 분석) → 독립적, 저장된 키워드와 연동 없음
- `/shopping` (쇼핑 인사이트) → 카테고리 코드 직접 입력 (사용자 불친절)
- `/ai` (AI 도구) → 3개 탭, 분석 결과와 연동 없음

### 페이지별 UX/UI 문제점

## 1. 저장된 키워드 (`/keywords/page.tsx`)

**현재 상태:** 카드 그리드 (1/2/3열), 각 카드에 메모/분석/삭제

| # | 문제점 | 심각도 | 설명 |
|---|--------|--------|------|
| 1-1 | 활용도가 낮은 단순 목록 | 높음 | "분석" 버튼 하나뿐. 비교/대량분석에 활용 불가 |
| 1-2 | 벌크 액션 없음 | 중간 | 선택 후 비교, 대량분석 전송 등 불가 |
| 1-3 | 정렬/필터 없음 | 중간 | 키워드가 많아지면 찾기 어려움 |
| 1-4 | 카드 정보량 부족 | 낮음 | 키워드명, 날짜, 메모만 표시 — 검색량 등 핵심 수치 없음 |

**개선 방향:**
- 독립 페이지 유지하되, **체크박스 선택 → "비교하기" / "대량 분석"** 액션 추가
- 각 카드에 마지막 분석 시 검색량/등급 요약 표시
- 검색/정렬(날짜, 이름순) 추가

---

## 2. 쇼핑 인사이트 (`/shopping/page.tsx`)

**현재 상태:** 카테고리 코드 텍스트 입력 + 필터 필(기간/기기/성별) + SVG 차트

| # | 문제점 | 심각도 | 설명 |
|---|--------|--------|------|
| 2-1 | 카테고리 코드 직접 입력 | **높음** | 일반 사용자가 네이버 카테고리 코드를 알 수 없음. UX 치명적 |
| 2-2 | 키워드 분석과 단절 | 중간 | 분석 결과에 쇼핑 데이터가 포함되지만 이 페이지와 연결 안됨 |
| 2-3 | 결과 비교 불가 | 낮음 | 한 번에 하나의 카테고리만 조회 가능 |
| 2-4 | PageHeader 미사용 | 낮음 | 다른 페이지와 달리 PageHeader 컴포넌트 미사용 |

**개선 방향:**
- 카테고리 코드 → **검색 가능한 드롭다운(Combobox)** 으로 교체 (주요 카테고리 프리셋 제공)
- 키워드 분석 결과에서 "쇼핑 트렌드 보기" 링크로 자연스러운 진입 경로 추가
- PageHeader 통일

## 3. 대량 분석 (`/bulk/page.tsx`)

**현재 상태:** 텍스트 입력/CSV 업로드 → 테이블 결과 + 엑셀 다운로드

| # | 문제점 | 심각도 | 설명 |
|---|--------|--------|------|
| 3-1 | 저장된 키워드 연동 없음 | 높음 | 매번 키워드를 직접 입력/업로드해야 함 |
| 3-2 | 결과에서 비교 진입 불가 | 중간 | 분석 결과 테이블에서 키워드 선택 → 비교 기능 없음 |
| 3-3 | "선택 저장" 버튼 비활성화 | 중간 | 기능이 아직 구현되지 않은 것으로 보임 |
| 3-4 | "AI 딥다이브" 버튼 비활성화 | 낮음 | 마찬가지로 미구현 |

**개선 방향:**
- 입력 영역에 **"저장된 키워드에서 가져오기"** 버튼 추가
- 결과 테이블 행 체크박스 → "선택 비교" 액션 추가
- 비활성 버튼들 구현 or 제거

## 4. 키워드 비교 (`/analyze/compare/page.tsx`) — **핵심 개선 대상**

**현재 상태:** 2~5개 키워드 수동 입력 → 비교 테이블 + 바 차트

| # | 문제점 | 심각도 | 설명 |
|---|--------|--------|------|
| 4-1 | 진입 경로가 사이드바뿐 | **높음** | 키워드 분석, 저장 목록, 대량 분석에서 바로 비교 불가 |
| 4-2 | 키워드를 항상 수동 입력 | **높음** | 이전에 분석한 키워드를 다시 타이핑해야 함 |
| 4-3 | 저장된 키워드 활용 불가 | 높음 | 저장된 키워드를 불러와서 비교하는 경로 없음 |
| 4-4 | 관련 키워드에서 바로 비교 불가 | 중간 | 분석 결과의 관련 키워드를 비교에 보낼 수 없음 |

**개선 방향 (3가지 접근법 중 선택):**

### Option A: URL 파라미터 + 다른 페이지에서 연결 (권장)
- `/compare?keywords=키워드1,키워드2` 형태로 URL 파라미터 지원
- 키워드 분석 결과에 "비교에 추가" 버튼 → 비교 페이지로 이동
- 저장된 키워드에서 체크박스 선택 → "비교하기" 버튼
- 대량 분석 결과에서 행 선택 → "비교하기" 버튼
- **장점:** 기존 페이지 구조 유지, 구현 간단
- **단점:** 페이지 이동 필요

### Option B: 키워드 분석 페이지 내 비교 탭/섹션 통합
- 키워드 분석 페이지에 "비교" 탭 또는 하단 섹션 추가
- 분석한 키워드가 자동으로 비교 대상에 추가됨
- **장점:** 페이지 이동 없이 즉시 비교
- **단점:** 분석 페이지가 이미 37KB로 크고 복잡함, 더 비대해질 위험

### Option C: 비교 페이지에 "저장된 키워드 불러오기" 추가
- 비교 페이지 입력란에 드롭다운/모달로 저장된 키워드 선택
- 최근 분석한 키워드 히스토리에서 선택
- **장점:** 비교 페이지 자체적으로 완결
- **단점:** 다른 페이지에서의 자연스러운 흐름은 여전히 없음

**추천: Option A + C 조합**
- URL 파라미터로 다른 페이지에서 키워드를 넘겨받고
- 비교 페이지 자체에서도 저장된 키워드를 불러올 수 있게

## 5. AI 도구 (`/ai/page.tsx`)

**현재 상태:** 3개 탭 (제목 추천, 글 초안, 콘텐츠 점수), 37.9KB 단일 파일

| # | 문제점 | 심각도 | 설명 |
|---|--------|--------|------|
| 5-1 | 키워드 분석과 단절 | 중간 | 분석한 키워드를 AI 도구에 바로 전달 불가 |
| 5-2 | 단일 파일 37.9KB | 낮음 | 유지보수성 — 3개 탭이 모두 한 파일에 있음 |
| 5-3 | 사용량 바 가독성 | 낮음 | 일간 제한 표시가 각 탭마다 반복됨 |

**개선 방향:**
- 키워드 분석 결과에 "AI 제목 추천" 바로가기 추가 (`/ai?keyword=xxx&tab=title`)
- 파일 분리 (tab별 컴포넌트화) — 유지보수 목적
- 사용량 바를 상단 1회 표시로 통합

## 종합 개선 계획 (구현 단계)

### Phase 1: 페이지 간 연결 강화 (핵심)

**1-A. 키워드 비교 페이지 URL 파라미터 지원**
- 파일: `src/app/(main)/analyze/compare/page.tsx`
- `useSearchParams`로 `?keywords=a,b,c` 파라미터 읽기
- 파라미터가 있으면 자동으로 입력 필드에 채우고 분석 시작
- 수용 기준: `/compare?keywords=키워드1,키워드2` 접속 시 자동으로 비교 실행

**1-B. 키워드 분석 결과에 액션 버튼 추가**
- 파일: `src/app/(main)/analyze/page.tsx`
- 결과 영역에 "비교에 추가" 버튼 → `/compare?keywords=현재키워드`로 이동
- 관련 키워드 각각에 "비교" 아이콘 버튼
- "AI 제목 추천" 바로가기 → `/ai?keyword=xxx&tab=title`
- "쇼핑 트렌드 보기" 바로가기 (쇼핑 데이터가 있는 경우)
- 수용 기준: 분석 결과에서 2클릭 이내로 비교/AI/쇼핑 페이지 진입

**1-C. AI 도구 페이지 URL 파라미터 지원**
- 파일: `src/app/(main)/ai/page.tsx`
- `?keyword=xxx&tab=title` 파라미터로 키워드 자동 입력 + 탭 전환
- 수용 기준: `/ai?keyword=테스트&tab=title` 접속 시 키워드 필드에 "테스트" 입력되고 제목 추천 탭 활성화

### Phase 2: 저장된 키워드 활용도 강화

**2-A. 저장된 키워드 페이지 벌크 액션**
- 파일: `src/app/(main)/keywords/page.tsx`
- 각 카드에 체크박스 추가
- 상단 툴바에 "선택 비교" (→ `/compare?keywords=...`), "대량 분석" (→ `/bulk`에 키워드 전달) 버튼
- 검색 필터 + 날짜순/이름순 정렬
- 수용 기준: 3개 키워드 선택 → "비교하기" 클릭 → 비교 페이지에 3개 키워드가 자동 입력됨

**2-B. 비교 페이지에 "저장된 키워드에서 불러오기"**
- 파일: `src/app/(main)/analyze/compare/page.tsx`
- 각 키워드 입력란 옆에 북마크 아이콘 → 클릭 시 저장된 키워드 드롭다운
- API: 기존 `GET /api/keywords/saved` 재활용
- 수용 기준: 비교 페이지에서 저장된 키워드를 드롭다운으로 선택 가능

**2-C. 대량 분석에 "저장된 키워드 가져오기"**
- 파일: `src/app/(main)/bulk/page.tsx`
- "직접 입력" / "CSV 업로드" 탭 옆에 3번째 탭 "저장된 키워드" 추가
- 체크박스로 선택 → 분석 대상에 추가
- 수용 기준: 저장된 키워드 탭에서 선택 후 "분석 시작" 가능

### Phase 3: 개별 페이지 UX 개선

**3-A. 쇼핑 인사이트 — 카테고리 입력 개선**
- 파일: `src/app/(main)/shopping/page.tsx`
- 카테고리 코드 텍스트 입력 → Combobox (검색 가능 드롭다운)으로 교체
- 주요 카테고리 프리셋 데이터 (패션의류, 식품, 디지털/가전 등) 제공
- 직접 입력도 여전히 가능 (고급 사용자용)
- PageHeader 컴포넌트 적용으로 디자인 통일
- 수용 기준: 카테고리를 검색/선택할 수 있는 드롭다운 제공

**3-B. 저장된 키워드 카드 정보 보강**
- 파일: `src/app/(main)/keywords/page.tsx`
- 카드에 마지막 분석 시 총 검색량, 등급 배지 표시
- API 변경 필요: `/api/keywords/saved` 응답에 `lastAnalysis` 필드 추가 (또는 프론트에서 별도 fetch)
- 수용 기준: 카드에 검색량 수치와 등급 배지가 표시됨

**3-C. 대량 분석 결과 → 비교 연결**
- 파일: `src/app/(main)/bulk/page.tsx`
- 결과 테이블 각 행에 체크박스 추가
- 상단 툴바에 "선택 비교" 버튼 → `/compare?keywords=...`
- 수용 기준: 결과에서 키워드 2~5개 선택 → 비교 페이지로 이동

### Phase 4: 코드 품질 & 디자인 통일

**4-A. AI 도구 파일 분리**
- `src/app/(main)/ai/page.tsx` (37.9KB) → 탭별 컴포넌트 분리
- `components/TitleTool.tsx`, `DraftTool.tsx`, `ScoreTool.tsx`
- 수용 기준: 동일한 기능, 파일당 < 15KB

**4-B. 중복 유틸리티 통합**
- `competitionBadgeClass()` 함수가 `analyze/page.tsx`와 `compare/page.tsx`에 중복
- `formatNumber()` 등 공통 유틸리티를 `src/shared/lib/` 로 추출
- 수용 기준: 중복 함수 0개

## 리스크 및 완화

| 리스크 | 영향 | 완화 방안 |
|--------|------|-----------|
| URL 파라미터 인코딩 (한글) | 키워드가 깨질 수 있음 | `encodeURIComponent` / `decodeURIComponent` 사용 |
| 저장된 키워드 API 추가 필드 | 백엔드 변경 필요 | 프론트에서 캐시된 데이터 활용 가능한지 먼저 확인 |
| 비교 페이지 자동 실행 | URL 파라미터로 접근 시 API 호출 발생 | 디바운스 + 사용량 체크 선행 |
| 쇼핑 카테고리 데이터 | 프리셋 목록 관리 필요 | 네이버 API 기반 주요 카테고리 100개 내외로 제한 |

## 검증 방법

- Phase 1 완료 후: 키워드 분석 → 비교 → AI 도구 플로우가 2클릭 이내로 가능한지 확인
- Phase 2 완료 후: 저장된 키워드에서 비교/대량분석까지 원클릭 진입 확인
- Phase 3 완료 후: 쇼핑 인사이트 카테고리 선택이 직관적인지 확인
- 전체: 다크모드 호환, 모바일 반응형 유지 확인
